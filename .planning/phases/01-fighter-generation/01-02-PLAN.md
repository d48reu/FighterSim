REWRITE `seed_fighters()` with this pipeline:

1. **Initialize RNGs** (per RESEARCH.md Pattern 4):
   - `py_rng = random.Random(seed)` for general randomness
   - `np_rng = np.random.default_rng(seed)` for stat distributions
   - `faker_instances = create_faker_instances(seed)`

2. **Compute per-weight-class quotas** using `allocate_archetypes()`:
   - New function: `allocate_archetypes(count_per_class: int, np_rng) -> list[str]`
   - Implements the pyramid rarity curve from CONTEXT.md:
     Journeyman ~30%, Gatekeeper ~25%, Phenom ~20%, Late Bloomer ~12%, Shooting Star ~8%, GOAT Candidate ~5%
   - Soft quotas with +/-2 fighter variance per archetype per class
   - Sequential allocation with Journeyman absorbing remainder
   - Returns shuffled list of archetype strings

3. **Assign career stages** per quota:
   - New function: `assign_career_stage(archetype: str, py_rng) -> str`
   - Returns one of: "prospect", "prime", "veteran", "transitional"
   - Overall target: 20% prospect, 35% prime, 25% veteran, 20% transitional
   - Archetype constraints (validity matrix per RESEARCH.md Pitfall 6):
     - GOAT Candidate: prime or veteran only (never prospect)
     - Late Bloomer: prime only (never prospect -- they bloom late)
     - Shooting Star: prime only (peak then decline)
     - Phenom: prospect or prime (young talent)
     - Gatekeeper: any stage
     - Journeyman: any stage
   - Age derived from career stage: prospect=20-24, prime=25-31, veteran=32-37, transitional=27-33

4. **Generate fighters** in a loop per weight class:
   - For each weight class, get ~90 fighters (450 total / 5 classes)
   - For each fighter slot:
     a. Get archetype from pre-allocated quota list
     b. Assign career stage (constrained by archetype)
     c. Derive age from career stage range using py_rng
     d. Pick nationality using `pick_nationality(py_rng)` from name_gen
     e. Generate name using `generate_name()` with uniqueness tracking
     f. Pick style using `py_rng.choice(list(FighterStyle))`
     g. Generate stats using `generate_stats(archetype, style, career_stage, np_rng)` from stat_gen
     h. Generate fight record using enhanced `_gen_record(age, career_stage, py_rng)`
     i. Set prime_start/prime_end based on career stage
     j. Assign traits using existing `_assign_traits()`
     k. Assign popularity/hype using existing `_starting_popularity_hype()`
     l. Assign nickname using `suggest_nicknames(fighter, session)` -- pick first suggestion
     m. Create Fighter row and add to session

5. **Distribute fighters to organizations** (prestige-gated per CONTEXT.md):
   - New function: `assign_organization(fighter, career_stage, archetype, orgs, py_rng) -> Organization | None`
   - Sort AI orgs by prestige descending
   - UCC (prestige 90): gets ~40% of prime and veteran fighters, ~10% of prospects
   - Bellator (prestige 70): gets ~25% of prime/veteran, ~20% of prospects
   - One Championship (prestige 75): gets ~25% of prime/veteran, ~20% of prospects
   - Player Promotion (prestige 50): gets ~10% of prime/veteran, ~30% of prospects, more journeymen/gatekeepers
   - Free agents: 10-15% of total (~45-68 fighters) -- weighted toward prospects and veterans, NO GOAT Candidates as free agents
   - For free agents: skip contract creation entirely
   - For signed fighters: create Contract with salary scaled to archetype (GOAT Candidates earn more)

6. **Keep `_adjust_record_for_archetype()`** -- call it after record generation for GOAT Candidates and Shooting Stars.

The function signature changes default count: `def seed_fighters(session, orgs, seed=42, count=450)`.
